01

n8n là gì?

Automation platform — Nối mọi thứ lại, chạy tự động 24/7

n8n (đọc là "n-eight-n") là công cụ automation workflow mã nguồn mở. Bạn kéo thả các "node" (khối chức năng) vào canvas, nối chúng lại → workflow tự chạy khi có sự kiện xảy ra.

Cài đặt & Khởi động

3 cách chạy n8n — chọn cái phù hợp với bạn

Cách 1 — n8n Cloud ✅ (Dễ nhất cho người mới)

Không cần cài gì, có ngay trong 2 phút. Phù hợp để học và test. 14 ngày dùng thử miễn phí.

1

Truy cập n8n.io → "Get started"

Đăng ký bằng email hoặc GitHub. Instance của bạn được tạo tự động tại yourname.n8n.cloud.

2

Tạo workflow đầu tiên

Click nút "+ New Workflow" → canvas trống xuất hiện → kéo thả nodes vào bắt đầu build.

3

Kích hoạt và chạy

Click "Execute Workflow" để test thủ công, hoặc toggle "Active" để workflow chạy tự động theo trigger.

Cách 2 — npx (Nhanh nhất trên máy local)
TerminalNode.js 18+ required
# Chạy n8n ngay, không cần cài đặt
npx n8n

# Mở browser tại:
http://localhost:5678

# Lần sau khởi động lại:
npx n8n start
📌Data được lưu trong ~/.n8n/ trên máy tính của bạn. Mỗi lần chạy npx n8n sẽ tiếp tục từ lần trước.
Cách 3 — Docker (Khuyến nghị cho production)
docker-compose.ymlDocker
version: '3.8'

services:
  n8n:
    image: docker.n8n.io/n8nio/n8n
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=your_password
      - GENERIC_TIMEZONE=Asia/Ho_Chi_Minh
    volumes:
      - ~/.n8n:/home/node/.n8n    # lưu data
    restart: unless-stopped
Terminal
docker compose up -d
# Truy cập: http://localhost:5678
💡Để học → dùng Cloud hoặc npx. Để chạy production ổn định 24/7 → dùng Docker trên VPS (DigitalOcean, Hetzner, Railway...).

Nodes — Khối xây dựng cơ bản

Mỗi node là một hành động — kết hợp lại thành workflow

Giải phẫu một Node
INPUT
📧
Gmail Node
Action · Google
OUTPUT
Input (chấm xanh — bên trái)Nhận dữ liệu từ node trước. Có thể từ 0 (trigger), 1, hoặc nhiều nguồn.
Output (chấm hồng — bên phải)Trả dữ liệu sang node tiếp theo. IF/Switch có nhiều output.
Body (phần thân)Click để mở panel cấu hình bên phải: chọn operation, điền tham số, kết nối credential.
Status (badge góc)Hiển thị ✅ thành công, ❌ lỗi, hoặc số items sau khi execute.
4 loại node chính
⚡
Trigger Nodes
Node đầu tiên. Xác định KHI NÀO workflow chạy. Ví dụ: Webhook, Schedule, Email nhận, Form submit...
🔧
Action Nodes
Thực hiện hành động trong app. Ví dụ: Gmail gửi email, Slack nhắn tin, Sheets ghi row...
🧩
Logic Nodes
Điều khiển luồng. IF, Switch, Merge, Loop, Wait, Error Trigger, Stop...
⚙️
Transform Nodes
Xử lý dữ liệu. Set, Edit Fields, Code Node, Remove Duplicates, Aggregate...
Cách thêm và cấu hình Node
1

Click + hoặc kéo từ output

Bấm dấu + trên canvas, hoặc kéo từ chấm tròn output của node hiện tại. Panel tìm kiếm node xuất hiện.

2

Tìm kiếm tên node

Gõ tên app: gmail, slack, sheets... hoặc gõ chức năng: if, http, code...

3

Chọn Operation

Mỗi node có nhiều operations. Ví dụ Gmail: Send Email, Get Emails, Reply... Chọn đúng operation cần dùng.

4

Điền tham số và kết nối Credential

Điền các field cần thiết. Nếu node cần xác thực (Gmail, Slack...), nhấn "Add Credential" để kết nối tài khoản.

5

Test Step

Nhấn "Test step" (hoặc Ctrl+Enter) để chạy riêng node đó và xem output ngay trong panel bên dưới.

Items & JSON Data Flow

Dữ liệu trong n8n là mảng JSON — mỗi phần tử là một "item"

Cấu trúc Items

Mọi dữ liệu di chuyển giữa các node đều ở dạng array of items. Mỗi item bắt buộc có field json chứa object dữ liệu, và có thể có field binary cho file.

📦 Cấu trúc chuẩn n8nn8n
// Dữ liệu giữa các node
[
  {
    json: {
      id: 1,
      name: "Nguyễn Minh",
      email: "minh@test.com",
      amount: 500000
    }
  },
  {
    json: {
      id: 2,
      name: "Trần Lan",
      email: "lan@test.com",
      amount: 1200000
    }
  }
]
⚙️ Đọc trong Code NodeJS
// Lấy tất cả items
const items = $input.all();

// Item đầu tiên
const first = $input.first();
first.json.name   // "Nguyễn Minh"
first.json.amount // 500000

// Duyệt qua tất cả
items.forEach(item => {
  console.log(item.json.name);
});
⚠️Quan trọng nhất: Mỗi node xử lý từng item một theo mặc định. Nếu 3 items vào Gmail node → Gmail gửi 3 email riêng biệt. Đây là behavior cốt lõi cần hiểu!

Expressions {{ }}

Dùng JavaScript mini trong bất kỳ field nào của node

Bất kỳ field nào trong n8n đều cho phép dùng {{ }} để viết biểu thức JavaScript. Click biểu tượng ⚡ Expression bên cạnh field để bật chế độ expression.

n8n Expressions — Cú pháp cơ bản
// ── Lấy field từ item hiện tại ──
{{ $json.name }}                     // Field đơn giản
{{ $json.address.city }}             // Nested object
{{ $json.tags[0] }}                  // Phần tử mảng đầu tiên

// ── Tính toán ──
{{ $json.price * $json.qty }}         // Phép nhân
{{ $json.score >= 90 ? 'A' : 'B' }} // Ternary

// ── Xử lý chuỗi ──
{{ $json.name.toUpperCase() }}        // In hoa
{{ $json.email.trim().toLowerCase() }} // Chuẩn hóa email
{{ `Xin chào ${$json.name}!` }}      // Template literal

// ── Ngày giờ (Luxon) ──
{{ $now.toISO() }}                    // ISO string hiện tại
{{ $now.toFormat('dd/MM/yyyy') }}    // Format ngày VN
{{ $today.toISODate() }}              // Chỉ ngày hôm nay
{{ $now.minus({ days: 7 }).toISO() }} // 7 ngày trước

// ── Lấy data từ node khác ──
{{ $('Set Node').first().json.total }}
{{ $('HTTP Request').all().length }}
Bảng biến đặc biệt trong Expression
BiếnLoạiÝ nghĩa
$jsonObjectDữ liệu của item đang xử lý (shorthand cho $input.item.json)
$input.all()ArrayTất cả items từ node trước
$input.first()ItemItem đầu tiên từ node trước
$('NodeName').first()ItemItem đầu tiên từ một node cụ thể theo tên
$nowDateTimeThời gian hiện tại (Luxon DateTime object)
$todayDateTimeNgày hôm nay (không có giờ phút giây)
$env.KEYStringBiến môi trường đã cấu hình trong Settings
$workflow.idStringID của workflow hiện tại
$execution.idStringID của lần chạy này (unique mỗi execution)

Cú pháp: phút giờ ngày tháng thứ
         min  hr  dom mon dow

Code Node — JavaScript tự do

Node mạnh nhất — viết bất kỳ logic nào bằng JS

Template Code Node chuẩn
Code Node — Cấu trúc cơ bảnJavaScript
// ── BƯỚC 1: Lấy dữ liệu từ node trước ──
const items = $input.all();    // Tất cả items
const first = $input.first(); // Chỉ item đầu

// ── BƯỚC 2: Xử lý dữ liệu ──
const results = items.map(item => {
  const { name, price, qty } = item.json;

  // Logic tùy ý...
  const total    = price * qty;
  const discount = total > 1000000 ? total * 0.1 : 0;
  const tier     = total > 2000000 ? 'Gold' : 'Silver';

  // ── BƯỚC 3: Return đúng cấu trúc ──
  return {
    json: {             // ← BẮT BUỘC phải có field 'json'
      ...item.json,    // giữ tất cả fields cũ
      total,           // thêm fields mới
      discount,
      tier,
      processedAt: new Date().toISOString(),
    }
  };
});

// ── BẮT BUỘC: return array of items ──
return results;
🔴Quy tắc vàng của Code Node: Return luôn là array, mỗi phần tử luôn là object có field json. Quên return hoặc sai format → node báo lỗi ngay!
Biến đặc biệt trong Code Node
Biến đặc biệt — n8n Built-ins
// ── Input ──
$input.all()                 // Array tất cả items
$input.first().json.name    // Truy cập field
$input.last()               // Item cuối
$input.all().length         // Số lượng items

// ── Lấy từ node cụ thể ──
$('Tên Node').all()         // Tất cả items từ node đó
$('Tên Node').first().json  // Item đầu từ node đó

// ── Thời gian (Luxon) ──
$now                         // DateTime hiện tại
$now.toISO()                 // ISO string
$now.toFormat('dd/MM/yyyy')  // Format tùy chỉnh
$now.minus({ days: 1 })       // Trừ 1 ngày

// ── Meta ──
$env.MY_KEY                   // Environment variable
$workflow.id                  // Workflow ID
$execution.id                 // Execution ID (unique)
Khi nào dùng Code Node thay vì Set Node?
📝
Dùng Set Node khi:
  • Thêm / đổi tên field đơn giản
  • Tính toán cơ bản: +, ×, chuỗi...
  • Format ngày với $now.toFormat()
  • Không cần loop, filter, reduce
⚙️
Dùng Code Node khi:
  • Cần filter, map, reduce
  • Logic phức tạp, nhiều điều kiện
  • Gộp/biến đổi nhiều items

Credentials — Kết nối tài khoản

Lưu trữ và quản lý thông tin xác thực an toàn

Credential là gì và cách tạo

Credential là nơi n8n lưu trữ API keys, OAuth tokens, username/password... một cách an toàn (được mã hóa). Tạo 1 lần, dùng lại trong nhiều workflow.

1

Mở Settings → Credentials

Trong giao diện n8n, vào menu bên trái → Credentials. Hoặc khi cấu hình node, click "Add Credential".

2

Chọn loại và nhập thông tin

Tìm tên service (Gmail, Slack, etc.). Làm theo hướng dẫn: có thể là OAuth2 (đăng nhập browser), API Key (copy paste), hoặc Basic Auth.

3

Test và Save

Click "Test connection" để kiểm tra. Sau khi save, credential xuất hiện trong dropdown của node tương ứng.

Schedule Trigger — Hàng ngày lúc 8h

2

Google Sheets — Đọc đơn hàng hôm qua

Thêm Google Sheets → Operation: Get Many → chọn spreadsheet/sheet → thêm Filter: date equals {{ $today.minus({days:1}).toISODate() }}. Thêm Credential Gmail/Google.

3

Code Node — Tính toán thống kê

Code Node
const rows = $input.all();

const total  = rows.reduce((s,i) => s + i.json.amount, 0);
const orders = rows.length;
const avg    = orders > 0 ? Math.round(total / orders) : 0;

return [{
  json: {
    total,
    orders,
    avg,
    totalFmt: total.toLocaleString('vi-VN') + 'đ'
  }
}];
4

Thêm field message, dùng Expression:

Expression trong Set Node
📊 *Báo cáo {{ $now.toFormat('dd/MM/yyyy') }}*

📦 Số đơn: *{{ $json.orders }}*
📈 TB/đơn: *{{ $json.avg.toLocaleString('vi-VN') }}đ*

Slack — Gửi thông báo
