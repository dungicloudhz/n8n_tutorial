JavaScript → n8n

Hiểu n8n thông qua ngôn ngữ bạn đã biết — mỗi node trên n8n đều tương ứng với một đoạn code JS quen thuộc

01
Variables & Dữ liệu
Cách n8n lưu trữ và truyền dữ liệu giữa các nodes
📦 Khai báo biến & Set Node ACTION
⚡ JavaScript
// Khai báo và gán biến
const name = "Nguyễn Văn A";
const age = 25;
const isActive = true;

// Object chứa nhiều fields
const user = {
  name: "Nguyễn Văn A",
  email: "a@gmail.com",
  score: 90
};

Output
🔴 n8n Node
Edit Fields (Set)
Set User Data
name= "Nguyễn Văn A"
email= "a@gmail.com"
score= 90
ModeKeep All Fields
💡
Trong n8n, node Edit Fields (Set) giống như gán biến trong JS. Dữ liệu truyền qua các node tiếp theo bằng $json.fieldName.
🔍 Truy cập dữ liệu từ node trước EXPRESSION
⚡ JavaScript
// Truy cập thuộc tính
user.name
user.email

// Nested object
response.data.items[0].id

// Destructuring
const { name, age } = user;
access.js
Output
🔴 n8n Node
Expression Syntax
Truy cập dữ liệu
Node hiện tại{{ $json.name }}
Nested{{ $json.data.items[0].id }}
Node khác{{ $node["Set User"].json.email }}
💡
n8n dùng {{ expression }} để truy cập dữ liệu. $json = output node trước, như response.data trong JS.

02
If / Switch → IF Node & Switch Node
Phân nhánh workflow theo điều kiện
🔀 If-Else → IF Node LOGIC
⚡ JavaScript
if (user.age >= 18) {
  // → nhánh TRUE
  sendEmail(user);
} else {
  // → nhánh FALSE
  blockUser(user);
}

if (score > 80 && isActive) {
  applyDiscount();
}
if-else.js
Output
🔴 n8n Node
IF Node
Check User Age
Condition$json.age >= 18
CombineAND
True branch→ Send Email
False branch→ Block User

IF node có 2 output ports: True và False

🎯 Switch-Case → Switch Node LOGIC
⚡ JavaScript
switch (order.status) {
  case "pending":
    sendConfirmation(); break;
  case "shipped":
    updateTracking(); break;
  case "delivered":
    markComplete(); break;
  default:
    logError();
}
switch.js
Output
🔴 n8n Node
Switch Node
Route By Order Status
ModeRules
Output 0status == "pending"
Output 1status == "shipped"
Output 2status == "delivered"
Fallback→ Log Error
💡
Switch node tạo nhiều output ports như các case. Dùng Expression mode để route theo {{ $json.status }}.
Vòng lặp & Mảng
03
Loops & Array Methods → SplitInBatches / Items
Xử lý danh sách dữ liệu
🔄 forEach / map → n8n Items CORE
⚡ JavaScript
// JS phải loop thủ công
const users = [
  { id: 1, name: "An" },
  { id: 2, name: "Bình" },
];

users.forEach(user => {
  sendEmail(user.email);
});

Output
🔴 n8n Node
Auto-loop (Built-in!)
🎉 Không cần viết loop

n8n tự làmChạy node cho TỪNG item

✅ Mỗi node tự chạy cho tất cả items!

💡
Điểm khác biệt lớn nhất: n8n tự động loop qua tất cả items. Không cần forEach — node sẽ chạy cho mỗi item trong mảng đầu vào.
🔧 Array .map() .filter() → Code Node TRANSFORM
⚡ JavaScript
// map: biến đổi từng phần tử
const emails = users.map(u => ({
  to: u.email,
  subject: `Hello ${u.name}`
}));

// filter: lọc phần tử
const adults = users.filter(
  u => u.age >= 18
);
array-methods.js
Output
🔴 n8n Node
Code Node (JS)
Map & Filter Items
ModeRun Once For All Items
Codereturn items.map(item => ({
json: { to: item.json.email,
subject: `Hello ${item.json.name}`
}}));
💡
Trong Code Node: items = array JS. Mỗi item có dạng { json: {...} }. Dùng mode "Run Once For All Items" để map/filter.
Functions & Modules
04
Functions → Code Node & Sub-workflows
Đóng gói logic tái sử dụng
⚙️ Function → Code Node CORE
⚡ JavaScript
function calculateDiscount(price, vip) {
  if (vip) return price * 0.8;
  if (price > 500000) return price * 0.9;
  return price;
}

const final = calculateDiscount(
  order.price, user.isVip
);

Output
🔴 n8n Node
Code Node
Calculate Discount
const price = $json.price;
const isVip = $json.isVip;
let final = isVip ? price*0.8 : price;
return [{ json: { ...\$json, finalPrice: final } }];

try {
  const result = await callAPI();

  console.error("Failed:", error);
  await sendSlackAlert({
    message: error.message
  });
}

🔁
n8n tự động loop
Không cần viết forEach. Mỗi node tự chạy cho tất cả items trong input — điểm khác biệt lớn nhất so với JS.
📦
items thay cho array
Trong Code Node, items = mảng JS. Mỗi item có cấu trúc { json: {...} } — luôn return đúng format này.
🔗
Expression = template literal
{{ $json.name }} trong n8n giống ${variable} trong JS template literal — dùng trong bất kỳ field nào.
🔑
Credentials thay cho env
API keys, tokens lưu trong Credentials giống .env file. Không hardcode vào workflow — bảo mật và tái sử dụng được.
🧵
Wire = data pipe
Kéo dây giữa nodes = function call chain trong JS. Data chạy từ trái sang phải.
🚀
Code Node = escape hatch
Khi không có node phù hợp, dùng Code Node viết JS thuần. $json, $items, $node là built-in variables.
