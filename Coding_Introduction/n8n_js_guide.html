JavaScript → n8n

Hiểu n8n thông qua ngôn ngữ bạn đã biết — mỗi node trên n8n đều tương ứng với một đoạn code JS quen thuộc

01
Variables & Dữ liệu
Cách n8n lưu trữ và truyền dữ liệu giữa các nodes
📦 Khai báo biến & Set Node ACTION Set / Edit Fields
⚡ JavaScript
// Khai báo và gán biến
const name = "Nguyễn Văn A";
const age = 25;
const isActive = true;

// Object chứa nhiều fields
const user = {
  name: "Nguyễn Văn A",
  email: "a@gmail.com",
  score: 90
};
🔴 n8n Node
Edit Fields (Set)
Set User Data
name = "Nguyễn Văn A"
email = "a@gmail.com"
score = 90
Mode Keep All Fields
💡
Trong n8n, node Edit Fields (Set) giống như việc gán biến trong JS. Bạn định nghĩa key-value, và n8n truyền chúng sang các node tiếp theo qua $json.fieldName. Chọn Keep All Fields để giữ dữ liệu cũ + thêm field mới, hoặc Replace để chỉ giữ field bạn set.
🔍 Truy cập dữ liệu từ node trước EXPRESSION $json / $node
⚡ JavaScript
// Truy cập thuộc tính object
user.name
user.email

// Truy cập nested object
response.data.items[0].id

// Destructuring
const { name, age } = user;

// Từ node khác (trong workflow)
// prevNode.output.data.field
🔴 n8n Expressions
Expression Syntax
Truy cập dữ liệu
Node hiện tại {{ $json.name }}
Nested {{ $json.data.items[0].id }}
Node cụ thể {{ $node["Set User"].json.email }}
All items {{ $items("HTTP Request") }}
💡
n8n dùng cú pháp {{ expression }} trong các field để truy cập dữ liệu. $json = dữ liệu từ node trước đó, như response.data trong JS. $node["Tên Node"].json giống như import result từ một function cụ thể.
Điều kiện & Phân nhánh
02
If / Switch → IF Node & Switch Node
Phân nhánh workflow theo điều kiện
🔀 If-Else → IF Node LOGIC IF Node
⚡ JavaScript
if (user.age >= 18) {
  // → nhánh TRUE
  sendEmail(user);
} else {
  // → nhánh FALSE
  blockUser(user);
}

// Kết hợp điều kiện
if (score > 80 && isActive) {
  applyDiscount();
}
🔴 n8n Node
IF Node
Check User Age
Condition 1 $json.age >= 18
Combine AND
True branch → Send Email
False branch → Block User

IF node có 2 output ports: True và False — kéo dây đến node tương ứng

🎯 Switch-Case → Switch Node LOGIC Switch Node
⚡ JavaScript
switch (order.status) {
  case "pending":
    sendConfirmation();
    break;
  case "shipped":
    updateTracking();
    break;
  case "delivered":
    markComplete();
    break;
  default:
    logError();
}
🔴 n8n Node
Switch Node
Route By Order Status
Mode Rules
Output 0 status == "pending"
Output 1 status == "shipped"
Output 2 status == "delivered"
Fallback → Log Error
💡
Switch node tạo nhiều output ports như các case trong JS. Mỗi rule là một điều kiện — data khớp rule nào sẽ đi ra port đó. Bạn có thể dùng Expression mode để route theo giá trị của field bất kỳ: {{ $json.status }}
Vòng lặp & Mảng
03
Loops & Array Methods → SplitInBatches / Items
Xử lý danh sách dữ liệu
🔄 forEach / map → n8n Items CORE Auto-loop mặc định
⚡ JavaScript
// JS phải loop thủ công
const users = [
  { id: 1, name: "An" },
  { id: 2, name: "Bình" },
];

users.forEach(user => {
  sendEmail(user.email);
});

// Hoặc for...of
for (const user of users) {
  callAPI(user.id);
}
🔴 n8n Auto-loop
n8n tự động loop!
🎉 Không cần viết loop
Input [ user1, user2, user3 ]
Node xử lý Send Email Node
n8n làm gì? Chạy node cho TỪNG item!

✅ Mỗi node trong n8n mặc định chạy cho tất cả items — tương đương forEach tự động!

📦 Batch Processing → SplitInBatches ACTION Split In Batches
⚡ JavaScript
// Chia mảng thành chunks
function chunk(arr, size) {
  const result = [];
  for (let i = 0; i < arr.length; i += size) {
    result.push(arr.slice(i, i + size));
  }
  return result;
}

// Xử lý 100 users, 10 cái/lần
const batches = chunk(users, 10);
for (const batch of batches) {
  await processBatch(batch);
  await delay(1000);
}
🔴 n8n Node
Split In Batches
Process in Batches of 10
Batch Size 10
Reset false
↓ loop back ↑
Wait Node
Delay 1 second
Wait Amount 1 second
🔧 Array .map() → Code Node / Item Lists TRANSFORM Code / Item Lists
⚡ JavaScript
// map: biến đổi từng phần tử
const emails = users.map(u => ({
  to: u.email,
  subject: `Hello ${u.name}`,
  sent: false
}));

// filter: lọc phần tử
const adults = users.filter(
  u => u.age >= 18
);
🔴 n8n Node
Code Node (JS)
Map & Transform Items
Mode Run Once For All Items
Code: return items.map(item => ({
  json: {
    to: item.json.email,
    subject: `Hello ${item.json.name}`
  }
}));
💡
Trong Code Node, mảng items = array JS của bạn. Mỗi item có dạng { json: {...} }. Để dùng .map(), chọn mode "Run Once For All Items" và return items.map(...). Để dùng như .forEach(), chọn mode "Run Once For Each Item".
Functions & Modules
04
Functions → Code Node & Sub-workflows
Đóng gói logic tái sử dụng
⚙️ Function → Code Node CORE Code Node
⚡ JavaScript
// Hàm xử lý dữ liệu
function calculateDiscount(price, vip) {
  if (vip) return price * 0.8;
  if (price > 500000) return price * 0.9;
  return price;
}

// Gọi hàm
const final = calculateDiscount(
  order.price,
  user.isVip
);
🔴 n8n Node
Code Node
Calculate Discount
const price = $json.price;
const isVip = $json.isVip;

let final = price;
if (isVip) final = price * 0.8;
else if (price > 500000) final = price * 0.9;

return [{ json: { ...$json, finalPrice: final } }];
🗂️ Module / Reusable Function → Sub-workflow ACTION Execute Workflow
⚡ JavaScript
// utils/sendNotification.js
export async function sendNotification(data) {
  await sendEmail(data);
  await sendSlack(data);
  await logDB(data);
}

// Gọi từ module khác
import { sendNotification } from "./utils";
await sendNotification({ user: user });
🔴 n8n Node
Execute Workflow
Call: Send Notification
Workflow ID WF_sendNotification
Wait for result true
Pass data {{ $json }}
💡
Sub-workflow = module/function tái sử dụng trong n8n. Tạo workflow với trigger When Called by Another Workflow, sau đó gọi bằng node Execute Workflow. Đây là cách tốt nhất để DRY (Don't Repeat Yourself) trong n8n, giống như extract một function dùng chung.

⚡ JavaScript
// Express webhook handler
app.post("/webhook/payment", async (req, res) => {
  const event = req.body;

  if (event.type === "payment.success") {
    await processOrder(event.data);
  }

  res.json({ received: true });
});
🔴 n8n Node
Webhook Trigger

Path payment-webhook

💡

Error Handling
08
Error Handling → Error Trigger & Try/Catch
Xử lý lỗi và retry trong workflow
🛡️ try/catch → Error Workflow TRIGGER Error Trigger
⚡ JavaScript
try {
  const result = await callAPI();
  await processData(result);
} catch (error) {
  // Ghi log lỗi
  console.error("Workflow failed:", error);

  // Gửi alert
  await sendSlackAlert({
    message: error.message,
    workflow: "payment"
  });
}
🔴 n8n Node
Error Trigger (Workflow riêng)
Handle Workflow Errors
Bắt lỗi từ $execution.error
Workflow $execution.workflow.name
Slack Node (sau Error Trigger)
Alert on Error
Message ❌ {{ $json.error.message }}
💡
n8n có 2 cách xử lý lỗi: (1) Error Workflow — tạo workflow riêng với trigger Error Trigger, set nó trong Settings của workflow chính như global catch block. (2) Node-level: bật "Continue On Fail" trong node options — giống try/catch chỉ cho một block nhỏ, data lỗi sẽ được pass tiếp với $json.error.

🔁
n8n tự động loop
Không cần viết forEach. Mỗi node tự chạy cho tất cả items trong input — đây là điểm khác biệt lớn nhất so với JS.
📦
items thay cho array
Trong Code Node, items = mảng JS. Mỗi item có cấu trúc { json: {...}, binary: {...} } — luôn return đúng format này.
🔗
Expression = template literal
{{ $json.name }} trong n8n giống như ${variable} trong JS template literal — dùng trong bất kỳ field nào.
🔑
Credentials thay cho env
API keys, tokens lưu trong Credentials (giống .env file). Không bao giờ hardcode vào workflow — tái sử dụng được và bảo mật hơn.
🧵
Wire = data pipe
Kéo dây giữa nodes = function call chain trong JS. Data chạy từ trái sang phải, mỗi node nhận output của node trước.
🚀
Code Node = escape hatch
Khi không có node phù hợp, dùng Code Node viết JS thuần. $json, $items, $node là built-in variables.
