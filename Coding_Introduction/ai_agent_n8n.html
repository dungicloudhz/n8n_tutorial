01

AI Agent trong n8n là gì?

Hiểu bản chất — Agent khác gì workflow thông thường?

AI Agent trong n8n là một workflow đặc biệt, trong đó LLM (Large Language Model) đóng vai trò "bộ não" — nó nhận đầu vào, suy nghĩ, lựa chọn công cụ phù hợp và tự thực thi. Không cần định nghĩa trước từng bước cụ thể như workflow thông thường.

⚙️ Workflow thông thường
  • Luồng chạy cố định A → B → C
  • Bạn tự định nghĩa từng bước
  • Không có khả năng "suy nghĩ"
  • Xử lý tốt khi quy trình rõ ràng
  • Dễ debug và kiểm soát
🤖 AI Agent Workflow
  • LLM tự quyết định bước tiếp theo
  • Tự chọn tool phù hợp với ngữ cảnh
  • Lặp lại cho đến khi hoàn thành mục tiêu
  • Xử lý tốt khi yêu cầu mơ hồ, phức tạp
  • Linh hoạt, thích nghi với tình huống mới
Kiến trúc ReAct của AI Agent

n8n AI Agent hoạt động theo vòng lặp ReAct (Reason + Act): LLM nhận câu hỏi, suy nghĩ (Thought), chọn hành động (Action), nhận kết quả (Observation), rồi lặp lại cho đến khi có câu trả lời cuối cùng.

Tools — Công cụ cho AI Agent

Agent chỉ mạnh khi được trang bị đúng tools — đây là "tay chân" của Agent

Tools là các hành động mà Agent có thể thực hiện. LLM sẽ tự quyết định khi nào gọi tool nào dựa vào mô tả của tool. Mô tả rõ ràng = Agent sử dụng tool chính xác hơn.

🌐
HTTP Request Tool
Gọi bất kỳ REST API nào. Agent tự điền params, headers theo ngữ cảnh
🔍
SerpAPI / Tavily
Tìm kiếm Google real-time. Rất hữu ích cho Agent cần tra thông tin mới nhất
📊
Google Sheets Tool
Đọc/ghi dữ liệu vào Google Sheets. Agent có thể tra cứu hoặc lưu thông tin
📧
Gmail / Outlook
Đọc email, tìm kiếm, gửi reply. Agent tự xử lý hộp thư của bạn
🗄️
SQL Database
Agent tự viết và chạy query SQL để tra cứu dữ liệu từ database
⚙️
Code Tool
Chạy đoạn JavaScript tùy ý. Tool mạnh nhất để xử lý logic phức tạp
🔗
Workflow Tool
Gọi một n8n workflow khác. Dùng để tạo Multi-Agent hoặc modular agents
📚
Vector Store Tool
Tìm kiếm trong knowledge base. Dùng cho RAG — trả lời câu hỏi theo tài liệu
🗂️
Notion / Airtable
Đọc/tạo/cập nhật records. Agent quản lý project, task, database của bạn

RAG & Vector Store

Cho Agent trả lời dựa trên tài liệu nội bộ của bạn — knowledge base riêng

RAG (Retrieval-Augmented Generation) là kỹ thuật cho phép LLM trả lời câu hỏi dựa trên tài liệu của bạn (PDF, Google Docs, website…) thay vì chỉ dựa vào training data. n8n tích hợp đầy đủ pipeline RAG.

🔄
Luồng RAG gồm 2 workflow:
1. Ingestion Workflow — Đọc tài liệu → Chunk → Embed → Lưu vào Vector Store (chạy 1 lần hoặc định kỳ)
2. Query Workflow — Nhận câu hỏi → Tìm kiếm semantic trong vector store → Đưa context vào LLM → Trả lời

You are an email processing assistant. Analyze each email and respond
with a JSON object ONLY (no other text):

{
  "category": "sales_inquiry | support | spam | invoice | other",
  "priority": "high | medium | low",
  "sentiment": "positive | neutral | negative",
  "summary": "Brief 1-sentence summary in Vietnamese",

Code Node — Parse JSON từ Agent outputJavaScript
const items = $input.all();
const results = [];

for (const item of items) {
  const agentOutput = item.json.output;

  try {
    // Tìm và parse JSON từ output của Agent
    const jsonMatch = agentOutput.match(/\{[\s\S]*\}/);
    const parsed = JSON.parse(jsonMatch[0]);

    results.push({
      json: {
        ...parsed,
        emailId: item.json.id,
        from: item.json.from,
        processedAt: $now.toISO()
      }
    });
  } catch (e) {
    results.push({ json: { error: 'Parse failed', raw: agentOutput } });
  }
}

return results;

Multi-Agent Workflow

Nhiều Agent chuyên biệt phối hợp — giải quyết bài toán phức tạp

Khi một bài toán quá phức tạp cho một Agent, bạn có thể tách thành nhiều Agent chuyên biệt. Agent điều phối (Orchestrator) phân công việc cho các Sub-Agent, mỗi agent làm tốt một nhiệm vụ cụ thể.

Multi-Agent Architecture — Orchestrator Pattern
💬
User Request
🎯
Orchestrator Agent
Phân tích & điều phối
🔍
Research Agent
Tìm kiếm thông tin
✍️
Writer Agent
Viết & chỉnh sửa
📊
Data Agent
Phân tích số liệu
✅
Tổng hợp kết quả
Final output to user
Cách triển khai: Workflow Tool

Mỗi Sub-Agent là một n8n workflow riêng biệt. Orchestrator gọi chúng thông qua Workflow Tool — giống như gọi một function.

1

Tạo Sub-Agent workflows riêng

Mỗi Sub-Agent là 1 workflow bắt đầu bằng Execute Workflow Trigger. Nhận input qua $json.query, trả output qua Set node cuối.

2

Trong Orchestrator: Thêm Workflow Tool

Thêm node "Execute Workflow" làm Tool, chọn workflow Sub-Agent. Viết description rõ ràng để LLM biết khi nào gọi Sub-Agent nào.

3

Orchestrator tự điều phối

Orchestrator Agent nhận request từ user, tự quyết định gọi Sub-Agent nào, tổng hợp kết quả và trả về câu trả lời cuối cùng.

⚠️
Lưu ý về chi phí: Multi-Agent gọi LLM nhiều lần hơn → tốn token hơn. Chỉ dùng khi bài toán thực sự cần phân công. Với hầu hết use case, một Agent đủ mạnh là đủ.

Prompt Engineering cho AI Agent

System prompt tốt = Agent hoạt động chính xác, ít lỗi hơn

Cấu trúc System Prompt chuẩn
Template — System Prompt cho AgentTemplate
## 1. IDENTITY — Vai trò của Agent
You are [tên và vai trò cụ thể], working for [công ty/ngữ cảnh].

## 2. OBJECTIVE — Mục tiêu chính
Your primary goal is to [mục tiêu cụ thể].

## 3. RULES — Quy tắc ứng xử
- Always respond in [ngôn ngữ]
- [Rule 1]
- [Rule 2]
- Never [điều cấm]

## 4. TOOL USAGE — Hướng dẫn dùng tools
- Use [tool_name] when [tình huống cụ thể]
- Always verify data with [tool] before responding

Tips viết prompt hiệu quả
✅
Nên làm
  • Viết bằng tiếng Anh cho system prompt
  • Chỉ định rõ format output (JSON, markdown…)
  • Đặt tên tool mô tả rõ chức năng
  • Thêm ví dụ cụ thể khi cần
  • Inject context động bằng expressions
❌
Không nên làm
  • Prompt quá dài và phức tạp
  • Thêm quá nhiều tool (Agent bị confused)
  • Không test với edge cases
  • Dùng model đắt tiền cho task đơn giản
  • Bỏ qua error handling

Các lỗi phổ biến và cách xử lý
Triệu chứngNguyên nhânCách sửa
Agent không dùng tool dù cần Tool description không rõ ràng Viết lại description rõ hơn, thêm "Use this when..."
Agent lặp vô tận, không có kết quả Max iterations quá cao, tool luôn trả error Giảm max iterations, fix lỗi trong tool được gọi

const agentResult = $input.first().json;

// Log toàn bộ intermediate steps
if (agentResult.intermediateSteps) {
  agentResult.intermediateSteps.forEach((step, i) => {
    console.log(`--- Step ${i + 1} ---`);
    console.log('Tool called:', step.action.tool);
    console.log('Input:', step.action.toolInput);
    console.log('Output:', step.observation);
  });
}

return [{ json: {
  finalAnswer: agentResult.output,
  stepsCount: agentResult.intermediateSteps?.length ?? 0,
  toolsUsed: agentResult.intermediateSteps
    ?.map(s => s.action.tool) ?? []
}}];
